# LSPD Binder

LSPD Binder — це зручний інструмент для створення макросів (біндів) та відображення зображень-підказок (шпаргалок) під час гри. Програма дозволяє автоматизувати введення тексту та натискання клавіш, а також виводити корисну інформацію безпосередньо поверх ігрового вікна.

Всі ваші налаштування та бінди зберігаються у файлі `Binds.txt` поруч із програмою. Ви можете редагувати їх як через вбудований графічний редактор біндів, так і вручну.

## 🌟 Основні можливості

- **Бінд-Дія**: Послідовне виконання команд (натискання клавіш, введення тексту, паузи).
- **Бінд-Картинка**: Відображення будь-яких зображень за посиланням поверх екрану гри (з підтримкою прозорості).
- **Меню вибору**: Якщо на одну клавішу призначено декілька біндів, з'явиться зручне меню вибору потрібного бінда.
- **Вбудований редактор**: Зручний інтерфейс для налаштування біндів без необхідності ручного редагування конфігу.

## 🎹 Де взяти назви клавіш

Для налаштування клавіш активації використовується синтаксис AutoHotkey v2. Повний список доступний за посиланням:
[Офіційний список клавіш AutoHotkey v2](https://www.autohotkey.com/docs/v2/KeyList.htm)

**Приклади назв клавіш:**
- `F1` .. `F12`
- `Enter`, `Tab`, `Escape`, `Space`, `Backspace`, `Delete`
- `Insert`, `Home`, `End`, `Left`, `Right`, `Up`, `Down`
- `Numpad0` .. `Numpad9`, `NumpadEnter`, `NumpadDot`
- `LButton`, `RButton`, `MButton` (кнопки миші)
- `A`..`Z`, `0`..`9`

## ⚡ Бінд-Дія (макроси)

Бінд-дія дозволяє створити послідовність кроків (натискання клавіш, вставлення тексту, затримки).

**Формат запису в `Binds.txt`:**
```text
[клавіша] Назва (необов'язково)
(кд_мс)              ; перезарядка (cooldown) в мілісекундах. За замовчуванням 1000 мс
KEY назва_клавіші    ; скрипт фізично натискає вказану клавішу
TEXT "ваш текст"     ; скрипт вставляє текст (лапки обов'язкові)
WAIT мілісекунди     ; пауза між кроками
```

**Приклад складної відігровки:**
```text
[F7] Арешт
(3000)
KEY Enter
TEXT "/me арештовує підозрюваного"
KEY Enter
WAIT 500
KEY U
TEXT "Оформлюю протокол, зачекайте"
KEY Enter
```

## 🖼️ Бінд-Картинка (шпаргалки поверх екрану)

Цей тип біндів дозволяє вивести картинку поверх гри при натисканні клавіші. Повторне натискання тієї ж клавіші приховує картинку.

**Формат запису в `Binds.txt`:**
```text
[клавіша] Назва (необов'язково)
IMG "пряме посилання на картинку"
(x, y, opacity, scale)
```

**Параметри позиціонування та вигляду:**
- `x`, `y` — відступ у пікселях від лівого верхнього кута екрана.
- `opacity` — рівень непрозорості від 0 (повністю невидимо) до 255 (повністю непрозоро).
- `scale` — масштаб у відсотках (100 = оригінальний розмір, 50 = половина розміру).

⚠️ **Важливо про фон картинки:**
Скрипт автоматично робить **ЧОРНИЙ колір (#000000)** прозорим — він буде невидимим поверх гри (ефект маски). Якщо фон картинки білий або іншого кольору, він залишиться видимим.

## ⌨️➕ Комбінації клавіш

Ви можете використовувати комбінації клавіш як для активації бінда, так і всередині кроків `KEY`.

**Модифікатори:** `Ctrl` (або `Control`), `Alt`, `Shift`, `Win` (або `Windows`). Регістр літер не має значення.

**Приклад:**
```text
[Ctrl+F6] Приклад
KEY Ctrl+V
KEY Alt+F4
```
*Примітка:* Символи `[` та `]` також можна використовувати в комбінаціях (наприклад, `KEY Ctrl+[`).

## 📋 Декілька біндів на одну клавішу

Якщо призначити кілька біндів на одну й ту саму клавішу, при її натисканні на екрані з'явиться зручне графічне меню вибору. 
- Бінди в меню активуються відповідними цифрами на клавіатурі від **1 до 9**. 
- 10-й бінд активується цифрою **0**.
- Максимальна кількість біндів на одну клавішу — 10. Більша кількість просто не буде відображатися.

## ⌨️ Перехоплення клавіш

При створенні бінда (або вручну у файлі `Binds.txt`) ви можете вказати, чи потрібно "блокувати" натискання клавіші для самої гри.

- **✅ «Не перехоплювати» (символ `~` перед клавішею, напр. `[~F2]`)**: Бінд спрацює, і гра ТЕЖ побачить, що ви натиснули цю клавішу.
- **❌ Звичайний режим (без `~`, напр. `[F2]`)**: Бінд спрацює, але гра НЕ отримає натискання цієї клавіші (воно буде повністю перехоплене біндером).

---

## 🛑 Екстрена зупинка та закриття

Клавіша `Esc` відіграє роль аварійної кнопки:
- Якщо довгий бінд-дія (наприклад, з кількома `WAIT`) вже виконується, натискання `Esc` **моментально зупинить** подальше виконання кроків.
- `Esc` також закриває всі відкриті картинки-шпаргалки та меню вибору біндів.
